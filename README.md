# «Анализ логов с целью выявления подозрительных событий»
Цель работы
Обработка и анализ журналов событий (WinEventLog и DNS) из набора данных botsv1.json для обнаружения потенциальных инцидентов информационной безопасности, а также визуализация наиболее часто встречающихся подозрительных событий.

Ход выполнения

Загрузка и подготовка данных
– Исходный файл botsv1.json загружен с использованием библиотеки pandas.
– Выполнена нормализация данных: приведение к единому формату, извлечение необходимых полей.

Разделение логов
– Сформированы два датафрейма: для событий Windows (WinEventLog) и для DNS-запросов.

Выявление подозрительных событий
– WinEventLog: использован перечень кодов событий, характерных для атак (неудачные входы – 4625, изменение прав – 4703, эскалация привилегий – 4672 и др.).
– DNS-логи: подозрительными признаны записи, содержащие в поле eventtype метку suspicious, а также доменные имена с признаками вредоносной активности (malicious, c2.).

Визуализация результатов
– Для объединённого набора подозрительных событий выполнен подсчёт частоты встречаемости.
– С помощью библиотек matplotlib и seaborn построена горизонтальная столбчатая диаграмма, отображающая десять наиболее распространённых подозрительных событий.

Полученные результаты
– Доминирующим подозрительным событием в проанализированных данных является изменение прав пользователя (EventID 4703), что может указывать на попытки повышения привилегий.
– В DNS-логах зафиксированы запросы к доменам, ассоциированным с командно-контрольными серверами (например, c2.maliciousdomain.com), что требует дополнительной проверки соответствующих хостов.

Выводы
Разработанный скрипт позволяет автоматизировать первичную обработку журналов событий и выделить индикаторы компрометации для последующего углублённого анализа. Результаты работы могут быть использованы при мониторинге информационной безопасности и реагировании на инциденты.
